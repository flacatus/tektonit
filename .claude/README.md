# Agent Skills - Source of Truth

This directory contains the **single source of truth** for tektonit's test generation knowledge. The agents and skills here are used in two ways:

## 1. Claude Code Integration (Human-Assisted Development)

When developers use Claude Code, these agents/skills are directly available:

```bash
# Interactive Claude Code can invoke skills directly
/generate-tekton-tests <catalog>
/diagnose-failure <test-file>
/fix-test <test-file>
```

The `.claude/` directory structure follows the [Agent Skills](https://agentskills.io/) open standard.

## 2. Container Agent (Autonomous Generation)

The containerized `tektonit` agent uses prompts generated FROM these skills:

```bash
# Build prompts.py from .claude/ agents
./scripts/sync-prompts.sh

# Then run container with generated prompts
tektonit generate <catalog>
```

## Workflow: Editing Agent Knowledge

### 1. Edit the Source (`.claude/` files)

```bash
# Edit agent or skill content
vim .claude/agents/stepaction-test-generator.md
```

These files use markdown with YAML frontmatter:

```markdown
---
name: stepaction-test-generator
description: |
  Specialist for Tekton StepActions.
  TRIGGER when: ...
  DO NOT TRIGGER when: ...
---

# Agent Content

Your agent knowledge here...
```

### 2. Regenerate Container Prompts

```bash
# Sync prompts.py from .claude/ source
./scripts/sync-prompts.sh
```

This reads `.claude/agents/*.md` and `.claude/skills/*.md` and generates `tektonit/prompts.py`.

### 3. Test the Changes

```bash
# Test the generated prompts
python -m pytest tests/test_prompts.py -v

# Test actual generation with new prompts
GEMINI_API_KEY=... tektonit generate-single <resource.yaml>
```

### 4. Commit Both

```bash
# Commit source and generated file together
git add .claude/ tektonit/prompts.py
git commit -m "Update agent knowledge: <what changed>"
```

## Key Principles

### Progressive Disclosure

Keep files under 500 lines. Main concepts in the main file, detailed examples inline:

```markdown
## Key Concept

**Brief explanation with WHY**

### Detailed Example
<code example>
```

### Explain WHY, Not Just HOW

```markdown
# Before (heavy-handed)
MUST embed script verbatim.
NEVER paraphrase.

# After (explanatory)
**Verbatim embedding:** Copy scripts character-for-character.
Why? Because even small paraphrasing changes behavior.
```

### TRIGGER Conditions

Every agent/skill description should include explicit trigger conditions:

```yaml
description: |
  What this agent does.

  TRIGGER when: user asks X, needs Y, wants Z
  DO NOT TRIGGER when: user asks A, already has B
```

### Decision-Oriented

Frame agents as decision-makers, not code generators:

```markdown
You are NOT just a code generator. You are an autonomous agent that:
- Makes judgments about priority
- Diagnoses root causes
- Chooses strategies
```

## Build Script Internals

`scripts/build_prompts_from_agents.py` does:

1. Reads `.claude/agents/*.md` and `.claude/skills/*.md`
2. Extracts markdown body (removes YAML frontmatter)
3. Converts markdown to plain text prompt format
4. Preserves existing template strings from `prompts.py`
5. Combines into `BATS_SYSTEM_PROMPT` and `PYTEST_SYSTEM_PROMPT`
6. Writes complete `tektonit/prompts.py`

The generated `prompts.py` has:
- Auto-generated header with timestamp
- Warning: "DO NOT EDIT THIS FILE DIRECTLY"
- System prompts from agents
- Preserved template strings
- All helper functions

## Validation

Tests ensure alignment between `.claude/` and `prompts.py`:

```bash
# Run validation tests
python -m pytest tests/test_prompts.py -v
```

Tests check:
- ✓ Key concepts present in prompts
- ✓ Explanatory style (not heavy-handed MUST/NEVER)
- ✓ Reasoning explanations included
- ✓ Template strings preserved
- ✓ Helper functions preserved
- ✓ Build script can regenerate successfully

## CI Integration

Add to `.github/workflows/` or similar:

```yaml
- name: Check prompts are in sync
  run: |
    ./scripts/sync-prompts.sh
    git diff --exit-code tektonit/prompts.py
```

This ensures developers don't edit `prompts.py` directly without regenerating from `.claude/`.
